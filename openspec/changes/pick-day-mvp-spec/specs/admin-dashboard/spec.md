## ADDED Requirements

### Requirement: 관리자 인증
관리자 링크(`/admin/{adminToken}`)에 접근할 때 관리자 비밀번호를 입력하여 인증한다. 세션 없이 각 접근마다 토큰+비밀번호로 검증한다.

#### Scenario: 올바른 관리자 비밀번호 입력
- **WHEN** 관리자가 `adminToken`과 일치하는 비밀번호를 입력하면
- **THEN** 관리자 대시보드가 표시된다

#### Scenario: 잘못된 관리자 비밀번호
- **WHEN** 잘못된 비밀번호를 입력하면
- **THEN** 에러 메시지를 표시하고 대시보드 접근을 차단한다

#### Scenario: 존재하지 않는 adminToken
- **WHEN** 유효하지 않은 `adminToken`으로 접근하면
- **THEN** 404 페이지를 표시한다

---

### Requirement: 응답 목록 조회
관리자는 해당 일정에 대한 모든 강사 응답을 리스트 형태로 조회할 수 있다.

#### Scenario: 응답 목록 표시
- **WHEN** 관리자가 대시보드에 접근하면
- **THEN** 강사명, 이메일, 전화번호, 선택 일정, 제출 시각을 포함한 응답 목록이 표시된다

#### Scenario: 응답 없는 경우
- **WHEN** 아직 응답이 없는 일정의 대시보드에 접근하면
- **THEN** 빈 상태 메시지를 표시한다

---

### Requirement: 일정별 응답 집계
관리자는 각 일정 후보에 대해 몇 명이 가능하다고 응답했는지 집계를 확인할 수 있다.

#### Scenario: 일정 후보별 집계 표시
- **WHEN** 관리자가 대시보드를 조회하면
- **THEN** 각 `schedule_option`에 대해 선택한 강사 수가 집계되어 표시된다

#### Scenario: 응답 수 기준 정렬
- **WHEN** 집계가 표시될 때
- **THEN** 응답 수가 많은 일정 후보부터 내림차순으로 정렬된다

---

### Requirement: 강사 배정
관리자는 응답한 강사 중 1명을 선택하여 특정 일정에 배정할 수 있다.

#### Scenario: 강사 배정
- **WHEN** 관리자가 특정 일정 후보와 강사를 선택하고 배정 버튼을 누르면
- **THEN** `responses` 레코드에 `assigned_option_id`가 저장된다

#### Scenario: 중복 배정 방지
- **WHEN** 이미 배정된 일정 후보에 다른 강사를 배정하면
- **THEN** 시스템은 기존 배정을 해제하고 새 강사를 배정한다 (동일 일정에 1명만 배정)
