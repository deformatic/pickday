# Pick Day

# 📌 Product Requirements Document (PRD)

## 1. 제품 개요 (Product Overview)

### 1.1 제품 정의

본 서비스는 교육 일정 조율을 위해
**“링크 기반 일정 수집 및 강사 매칭 시스템”**이다.

* 회원가입 없이 사용 가능
* 링크 공유만으로 일정 응답 수집
* 강사 풀 내 일정 매칭 지원

---

## 1.2 해결 문제

기존 문제:

* 메신저/전화 기반 일정 조율
* 반복 커뮤니케이션
* 일정 충돌 및 누락
* 기록 부재

해결 방식:

> 일정 조율을 “대화 → 데이터 입력” 구조로 전환

---

## 1.3 핵심 가치

* 커뮤니케이션 비용 감소
* 응답 속도 증가
* 일정 정확도 향상
* 관리 효율성 개선

---

## 3. 핵심 기능 (Core Features)

---

### 3.1 일정 생성

#### 입력 항목

* 교육명
* 지역
* 시간 정보
* 일정 후보 (복수)

#### 옵션

* 비밀번호 설정 (선택)
* 관리자 비밀번호 설정 (필수)
* 강사 이메일 필수 여부 (선택, 기본값: 선택)
* 강사 전화번호 필수 여부 (선택, 기본값: 선택)

> 관리자는 링크 생성 시 이메일과 전화번호 각각에 대해 응답 필수 여부를 독립적으로 설정할 수 있다.

---

### 3.2 링크 생성

```plaintext
참여 링크: /s/{scheduleToken}
관리자 링크: /admin/{adminToken}
```

---

### 3.3 접근 모드

#### ① Public 모드

* 비밀번호 없음
* 누구나 접근 가능

#### ② Protected 모드

* 비밀번호 필요
* 인증 후 접근

---

### 3.4 강사 응답 기능

#### 입력 항목

| 항목 | 기본 | 관리자 설정 시 |
|------|------|--------------|
| 강사명 | 필수 | 항상 필수 |
| 이메일 | 선택 | 필수로 변경 가능 |
| 연락처 | 선택 | 필수로 변경 가능 |
| 가능 일정 | 필수 (복수 선택) | — |
| 코멘트 | 선택 | — |

> 이메일/연락처의 필수 여부는 관리자가 링크 생성 시 설정하며, 응답 폼에 동적으로 반영된다.

---

### 3.5 자동완성 (중복 방지)

#### 기능

* 이름/이메일/연락처 기반 검색
* 기존 입력 이력 불러오기

#### InstructorIdentity 등록 방식

> 강사가 처음 응답할 때 자동으로 `InstructorIdentity`에 등록된다.
> 관리자가 별도로 강사 목록을 사전 등록하지 않는다.

```plaintext
강사 응답 제출 → InstructorIdentity 존재 여부 확인
  → 없음: 신규 생성 후 Response에 연결
  → 있음: 기존 레코드에 연결 (이름+이메일+연락처 기준 매칭)
```

#### UX

```plaintext
입력 → 추천 리스트 표시 → 사용자 선택
```

---

### 3.6 응답 수정

* 기존 입력 수정 가능
* 동일 사용자 재입력 허용

---

### 3.7 관리자 기능

* 응답 리스트 조회
* 일정별 집계
* 강사 선택/배정
